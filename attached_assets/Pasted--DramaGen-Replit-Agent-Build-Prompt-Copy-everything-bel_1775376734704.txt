## What to Build

Build **DramaGen** — a web app that transforms your calm, tired, low-energy text into dramatic, over-the-top scolding audio clips spoken in your own cloned voice (or a designed dramatic character voice). Think Key & Peele's "Anger Translator" meets Kagi's LinkedIn Translator, but for generating downloadable audio rants.

**The tagline:** "Too tired to yell? Let DramaGen do it for you."

**The core loop:**
1. User types a mild message (e.g., "Please stop leaving dishes in the sink")
2. Picks a drama mode (Rage, Passive-Aggressive, Telenovela, Disappointed Parent, TikTok Fight Starter, etc.)
3. AI translates the text into a dramatic version in that style
4. The dramatic text is converted to speech using ElevenLabs — either the user's **cloned voice** or a **designed character voice**
5. User previews, then downloads the audio clip to send to whoever needs to hear it

---

## Tech Stack

- **Frontend:** React + TypeScript + Tailwind CSS + shadcn/ui components
- **Backend:** Node.js + Express API server
- **AI Text Translation:** OpenAI API (GPT-4o-mini) for transforming mild text into dramatic text
- **Voice Cloning:** ElevenLabs Instant Voice Cloning (IVC) API — user records ~30s of audio in-browser, uploads to ElevenLabs, gets a voice clone
- **Voice Design:** ElevenLabs Voice Design API — pre-built dramatic character voices generated from text descriptions
- **Text-to-Speech:** ElevenLabs TTS API (eleven_flash_v2_5 model) — generates the final audio clip
- **Audio Recording:** Browser MediaRecorder API for capturing user voice samples
- **Database:** Replit's built-in PostgreSQL (or SQLite) for storing generated clips and voice IDs
- **File Storage:** Store generated audio clips as files on the server for download

## Backend API Endpoints

### `POST /api/translate`
Transforms mild text into dramatic text using OpenAI.

**Request body:**
```json
{
  "text": "Please do your dishes",
  "mode": "rage"
}
```

**Implementation:**
Use OpenAI's API with a carefully crafted system prompt for each drama mode. The system prompt should:
- Transform the input into the selected style
- Keep it funny and entertaining, never genuinely harmful or threatening
- Add dramatic flair, emphasis markers (ALL CAPS for yelling, *pauses* for dramatic effect)
- Keep output under 800 characters (good for TTS)
- Include stage directions in brackets for the TTS to interpret naturally

**System prompts per mode (use these exactly):**

**Rage Mode:**
```
You are an anger translator. The user will give you a calm, mild message. Your job is to translate it into a full-blown, over-the-top angry rant as if the speaker has completely lost it. Use ALL CAPS for emphasis, dramatic exaggeration, rhetorical questions, and comedic fury. Keep it funny and entertaining — never genuinely threatening or hateful. Think comedy roast meets Gordon Ramsay meltdown. Max 600 characters. Output ONLY the translated rant, nothing else.
```

**Passive Aggressive:**
```
You are a passive-aggressive translator. The user will give you a calm message. Transform it into the most devastatingly passive-aggressive version possible. Use fake politeness, backhanded compliments, "just so you know" energy, and weaponized kindness. Think smiling through gritted teeth. Sprinkle in phrases like "No worries at all!", "It's totally fine", "I just think it's funny how...", "But you do you!". Make it hilariously cutting while maintaining a veneer of politeness. Max 600 characters. Output ONLY the translated text.
```

**Disappointed Parent:**
```
You are a disappointed parent translator. The user gives you a calm message. Transform it into something a deeply disappointed parent would say — the kind that makes you feel worse than yelling ever could. Use guilt trips, "I'm not mad, just disappointed" energy, references to sacrifice, heavy sighs (written as "..."), and emotional devastation. Make it funny but painfully relatable. Max 600 characters. Output ONLY the translated text.
```

**Telenovela:**
```
You are a telenovela drama translator. Transform the user's calm message into an outrageously dramatic telenovela monologue. Include dramatic pauses (...), gasps, references to betrayal, destiny, and honor. Speak as if revealing a devastating plot twist. Over-the-top theatrical emotion. Think "¡No puede ser! After everything I've done for you!" energy, but in English. Max 600 characters. Output ONLY the translated monologue.
```

**TikTok Fight Starter:**
```
You are a TikTok drama translator. Transform the user's calm message into a viral TikTok-style confrontation script. Use Gen-Z slang, "girl..." energy, "the audacity", "bestie let me tell you", "no because WHY", nail painting emoji energy 💅, and the format of those viral "if your man does X" videos. Keep it fun, sassy, and meme-worthy. Max 600 characters. Output ONLY the translated text.
```

**Drill Sergeant:**
```
You are a military drill sergeant translator. Transform the user's calm message into a full drill sergeant tirade. ALL CAPS yelling, commands, rhetorical questions, "DID I STUTTER?!", push-up threats, comparing the person to a wet noodle. Pure over-the-top boot camp energy. Funny and absurd, not actually military. Max 600 characters. Output ONLY the translated text.
```

**Corporate Fury:**
```
You are a corporate rage translator. Transform the user's calm message into the most devastating corporate-speak takedown. Use "per my last email" energy, CC threats, mentions of "aligning on expectations", "circling back", "as previously communicated", implied career consequences, and weaponized professionalism. The rage is real but expressed entirely through HR-safe language. Max 600 characters. Output ONLY the translated text.
```

**Ice Cold:**
```
You are an ice-cold intimidation translator. Transform the user's calm message into something terrifyingly calm and cold. Short sentences. Deadly quiet. No yelling — that's what makes it scary. Think villain monologue energy. Calculated, measured, with an underlying threat that's never explicitly stated. Use pauses (...) and deliberate word choices. Scarier than any rant. Max 600 characters. Output ONLY the translated text.
```

**Response:**
```json
{
  "original": "Please do your dishes",
  "translated": "OH SO WE'RE JUST LIVING IN A ZOO NOW? ...",
  "mode": "rage"
}
```

## Database Schema

```sql
CREATE TABLE generations (
  id SERIAL PRIMARY KEY,
  original_text TEXT NOT NULL,
  translated_text TEXT NOT NULL,
  mode VARCHAR(50) NOT NULL,
  voice_type VARCHAR(20) NOT NULL, -- 'clone' or 'preset'
  voice_name VARCHAR(100),
  audio_filename VARCHAR(255),
  is_public BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE preset_voices (
  id VARCHAR(50) PRIMARY KEY,
  elevenlabs_voice_id VARCHAR(100) NOT NULL,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  emoji VARCHAR(10),
  preview_filename VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE reactions (
  id SERIAL PRIMARY KEY,
  generation_id INTEGER REFERENCES generations(id),
  emoji VARCHAR(10) NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## Key Implementation Details

### Browser Audio Recording
Use the MediaRecorder API to record the user's voice:
```javascript
// Request microphone access
const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
const mediaRecorder = new MediaRecorder(stream, { mimeType: 'audio/webm' });
const chunks = [];

mediaRecorder.ondataavailable = (e) => chunks.push(e.data);
mediaRecorder.onstop = () => {
  const blob = new Blob(chunks, { type: 'audio/webm' });
  // Send blob to /api/voice/clone
};

// Start recording
mediaRecorder.start();

// Stop after 30 seconds
setTimeout(() => mediaRecorder.stop(), 30000);
```

### ElevenLabs Voice Cloning (Server-side)
```javascript
const FormData = require('form-data');
const fetch = require('node-fetch');

async function cloneVoice(audioBuffer, name) {
  const form = new FormData();
  form.append('name', name);
  form.append('files', audioBuffer, { filename: 'recording.webm', contentType: 'audio/webm' });
  form.append('remove_background_noise', 'true');
  form.append('description', 'User voice clone for DramaGen');

  const response = await fetch('https://api.elevenlabs.io/v1/voices/add', {
    method: 'POST',
    headers: { 'xi-api-key': process.env.ELEVENLABS_API_KEY },
    body: form
  });
  
  const data = await response.json();
  return data.voice_id;
}
```

### ElevenLabs Voice Design (Server-side, run at startup)
```javascript
async function designVoice(description, sampleText) {
  const response = await fetch('https://api.elevenlabs.io/v1/text-to-voice/design', {
    method: 'POST',
    headers: {
      'xi-api-key': process.env.ELEVENLABS_API_KEY,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      model_id: 'eleven_multilingual_ttv_v2',
      voice_description: description,
      text: sampleText
    })
  });
  
  const data = await response.json();
  const preview = data.previews[0];
  
  // Save the voice permanently
  const createResponse = await fetch('https://api.elevenlabs.io/v1/text-to-voice/create', {
    method: 'POST',
    headers: {
      'xi-api-key': process.env.ELEVENLABS_API_KEY,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      voice_name: 'DramaGen - ' + name,
      voice_description: description,
      generated_voice_id: preview.generated_voice_id
    })
  });
  
  return createResponse.json(); // { voice_id: "..." }
}
```

### ElevenLabs TTS Generation (Server-side)
```javascript
async function generateSpeech(text, voiceId, voiceSettings) {
  const response = await fetch(
    `https://api.elevenlabs.io/v1/text-to-speech/${voiceId}`,
    {
      method: 'POST',
      headers: {
        'xi-api-key': process.env.ELEVENLABS_API_KEY,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        text: text,
        model_id: 'eleven_flash_v2_5',
        voice_settings: voiceSettings
      })
    }
  );
  
  const audioBuffer = await response.arrayBuffer();
  // Save to file and return URL
  const filename = `drama_${Date.now()}.mp3`;
  fs.writeFileSync(`./public/audio/${filename}`, Buffer.from(audioBuffer));
  return filename;
}
```

## Full User Flow

1. User lands on `/` — sees the fun landing page, clicks "Start Generating Drama"
2. Taken to `/generate`
3. Types their mild message in the textarea (or clicks a quick-fill example)
4. Selects a drama mode card (e.g., "Rage Mode 🔥")
5. Chooses voice:
   - Option A: Clicks "Clone My Voice" tab → records 30s of audio → clicks "Clone" → waits 5-10s → voice cloned ✅
   - Option B: Clicks "Pick a Character Voice" tab → previews voices → selects one
6. Clicks "Generate Drama 🎬"
7. Backend: translates text via OpenAI → generates TTS via ElevenLabs → returns audio URL
8. UI shows: original vs. dramatic text side-by-side, audio player with the clip
9. User clicks "Download" → gets .mp3 file
10. Optionally clicks "Share to Gallery" to make it public
11. Can browse `/gallery` to see/hear other people's generations

## Important Notes

- **Keep it fun, never harmful.** The AI translations should be comedic and over-the-top, never genuinely threatening, hateful, or targeting protected groups. Add a content moderation check on the input.
- **Voice clone consent.** Show a brief consent notice before recording: "By recording your voice, you consent to creating an AI voice clone for use within DramaGen."
- **Rate limiting.** Add basic rate limiting (e.g., 10 generations per hour per IP) to avoid API cost runaway.
- **Error handling.** Handle ElevenLabs API errors gracefully — show a funny error message like "Even DramaGen needs a moment to cool down. Try again!"
- **Audio cleanup.** Delete generated audio files older than 24 hours to manage storage (unless saved to gallery).
- **Mobile-responsive.** The app MUST work well on mobile since people will use it on their phones.
- **Loading states.** The generation takes 5-15 seconds. Use engaging loading animations with rotating funny messages.
